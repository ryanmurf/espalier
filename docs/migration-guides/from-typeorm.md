# Migrating from TypeORM to Espalier

## Why Migrate?

- **Standard decorators**: Espalier uses TC39 Stage 3 decorators — no `experimentalDecorators` flag needed
- **Active maintenance**: Regular quarterly releases with new features
- **Multi-runtime**: Works on Node.js, Bun, Deno, and Cloudflare Workers
- **Built-in tooling**: Studio, testing utilities, migration generation

## Entity Mapping

### TypeORM

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne } from "typeorm";

@Entity()
export class User {
  @PrimaryGeneratedColumn("uuid")
  id: string;

  @Column()
  name: string;

  @Column({ unique: true })
  email: string;

  @ManyToOne(() => Organization, org => org.users)
  organization: Organization;
}
```

### Espalier

```typescript
import { Entity, Id, Column, Table, ManyToOne } from "espalier-data";

@Entity()
@Table("user")
class User {
  @Id()
  @Column()
  id!: string;

  @Column()
  name!: string;

  @Column({ unique: true })
  email!: string;

  @ManyToOne(() => Organization)
  organization!: Organization;
}
```

### Key Differences

| TypeORM | Espalier | Notes |
|---------|----------|-------|
| `@PrimaryGeneratedColumn()` | `@Id() @Column()` | Espalier separates PK marking from column definition |
| `@Column({ nullable: true })` | `@Column({ nullable: true })` | Same syntax |
| `@CreateDateColumn()` | `@Column({ type: "TIMESTAMPTZ" })` | Use lifecycle hooks for auto-set |
| `@VersionColumn()` | `@Version()` | Same purpose, different decorator |
| `@Entity("table_name")` | `@Entity() @Table("table_name")` | Separated for clarity |

## Repository Pattern

### TypeORM

```typescript
const userRepo = dataSource.getRepository(User);
const users = await userRepo.find({ where: { active: true } });
const user = await userRepo.findOneBy({ id: "123" });
```

### Espalier

```typescript
interface UserRepository {
  findById(id: string): Promise<User | null>;
  findByActive(active: boolean): Promise<User[]>;
}

const userRepo = createRepository<User, UserRepository>(User);
const users = await userRepo.findByActive(conn, true);
const user = await userRepo.findById(conn, "123");
```

## Migrations

### TypeORM

```typescript
// typeorm migration:generate -n CreateUsers
export class CreateUsers1234567890 implements MigrationInterface {
  async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`CREATE TABLE "user" (...)`);
  }
  async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`DROP TABLE "user"`);
  }
}
```

### Espalier

```typescript
// Generated by: espalier schema generate
import type { Migration } from "espalier-data";

const migration: Migration = {
  version: "20260305120000",
  description: "create_users",
  up() {
    return [`CREATE TABLE "user" (...)`];
  },
  down() {
    return [`DROP TABLE "user"`];
  },
};

export default migration;
```

## Step-by-Step Migration

1. **Install Espalier** alongside TypeORM (they can coexist)
2. **Convert entities** one at a time — start with leaf entities (no relations)
3. **Create Espalier repositories** for converted entities
4. **Update service layer** to use Espalier repositories
5. **Migrate relations** — convert `@ManyToOne`, `@OneToMany`, `@ManyToMany`
6. **Switch data source** — replace TypeORM DataSource with Espalier's
7. **Remove TypeORM** once all entities are converted

## Common Patterns

### Query Builder

```typescript
// TypeORM
const users = await userRepo
  .createQueryBuilder("user")
  .where("user.age > :age", { age: 18 })
  .orderBy("user.name")
  .getMany();

// Espalier — use Specifications
const adults = where("age", "gt", 18);
const users = await userRepo.findAll(conn, adults, { orderBy: "name" });
```

### Transactions

```typescript
// TypeORM
await dataSource.transaction(async manager => {
  await manager.save(user);
  await manager.save(order);
});

// Espalier
const conn = await dataSource.getConnection();
const tx = await conn.beginTransaction();
try {
  await userRepo.save(conn, user);
  await orderRepo.save(conn, order);
  await tx.commit();
} catch (e) {
  await tx.rollback();
  throw e;
} finally {
  await conn.close();
}
```
